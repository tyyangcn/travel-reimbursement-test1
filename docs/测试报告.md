# 差旅报销管理系统测试报告

## 1. 测试目的

验证差旅报销系统后端主要接口是否能够正确处理报销单、行程、补助和费用分摊业务，并检查参数校验、状态控制、金额计算和关联数据同步是否符合设计要求。

## 2. 测试环境

| 项目 | 配置 |
| --- | --- |
| 操作系统 | Windows |
| JDK | Java 17 |
| 后端框架 | Spring Boot 3.2.12 |
| 数据访问 | MyBatis-Plus 3.5.12 |
| 数据库 | MySQL 8.4 |
| 构建工具 | Maven 3.9 |
| 接口工具 | Postman |
| 服务地址 | `http://localhost:8082` |
| 数据库名称 | `travel_reimbursement` |

## 3. 测试范围

- 报销单分页查询、详情查询、新增、修改和送审
- 行程新增、查询、修改和删除
- 补助及补助日历自动生成
- 补助日历勾选、实际金额调整和金额范围校验
- 行程变化后的补助重新计算
- 费用分摊保存、查询和删除
- 分摊首行自动补足、均摊和合计校验
- 报销单补助总金额汇总
- 参数校验、业务异常和统一响应

## 6. 测试结论

本轮 Postman 接口测试覆盖了后端主要业务流程。报销单、行程、补助和费用分摊功能能够正常运行，行程与补助数据能够保持关联，补助金额可以正确汇总到报销单主表，费用分摊比例和金额计算结果正确。

当前已建立 11 个 JUnit 自动化测试，其中包括 1 个 Spring Boot 上下文测试、7 个报销单业务测试和 3 个补助调整业务测试，测试结果为 11 个全部通过。前端已完成 TypeScript 类型检查，但仍需在浏览器中执行完整业务流程回归、兼容性测试和最终 UI 像素级验收。

## 7. 后续测试建议

- 增加报销单不存在、行程不存在等异常用例。
- 增加审批中单据禁止修改、删除和重复送审的状态测试。
- 增加日期范围错误、分摊比例不等于 100% 等边界测试。
- 继续使用 MockMvc 补充 Controller 层 HTTP 状态码和响应结构测试。
- 执行新增、编辑、送审、补助和分摊的浏览器端到端测试。
